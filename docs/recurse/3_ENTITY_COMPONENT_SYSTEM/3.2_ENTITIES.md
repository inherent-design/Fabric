## Entity Instantiation Rules

### Component Dependencies
1. **All Entities**: Must have PositionComponent and ProcessingLevelComponent
2. **Physical Entities**: Require PhysicsComponent and BoundsComponent  
3. **Thinking Entities**: Require AIBrainComponent and SensoryComponent
4. **Function Users**: Require FunctionCapabilityComponent
5. **Faction Members**: Require FactionComponent

### Performance Optimization
- **Tier 0-1 Entities**: BACKGROUND or MINIMAL processing when distant
- **Tier 2-3 Entities**: REDUCED processing when not in active combat
- **Tier 4+ Entities**: Always FULL processing (boss-level entities)
- **World Objects**: Generally BACKGROUND unless interacting

### Dynamic Component Addition
Some entities can gain/lose components during gameplay:
- **Parasites** can add components to infected entities
- **Glitches** can corrupt components in other entities  
- **Memory Fragments** can be consumed (entity destroyed)
- **Players** can unlock new function capabilities

This entity system creates a diverse ecosystem where every entity's behavior emerges from its component composition, enabling complex interactions and emergent gameplay scenarios.
