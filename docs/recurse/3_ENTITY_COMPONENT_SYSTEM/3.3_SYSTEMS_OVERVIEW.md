# RECURSE ECS Systems Overview
*Data Flow and System Dependencies*

This document provides a high-level overview of all ECS systems in RECURSE, defining their responsibilities and data dependencies. Each system reads from and writes to specific components, creating a clear data flow architecture.

---

## System Architecture Principles

**Single Responsibility**: Each system handles one specific aspect of game logic
**Clear Dependencies**: Systems declare exactly which components they read/write
**Parallel Processing**: Systems with non-overlapping write components can run in parallel
**Hierarchical Updates**: Critical systems run before dependent systems

---

## Core Simulation Systems

### WorldStateSystem
**Responsibility**: Manages global reality stability and temporal state
**Update Frequency**: Every frame
**Reads**: All FunctionExecutorComponent, RealityAnchorComponent, EffectComponent
**Writes**: WorldState (global), ProcessingLevelComponent (stability-based LOD)
**Dependencies**: None (runs first)

### StabilitySystem  
**Responsibility**: Calculates and applies stability changes from function usage
**Update Frequency**: Every frame
**Reads**: FunctionExecutorComponent, RealityAnchorComponent, TemporalComponent
**Writes**: WorldState.stability, EffectComponent (stability effects)
**Dependencies**: WorldStateSystem

### TemporalSystem
**Responsibility**: Processes time dilation effects and causality chains
**Update Frequency**: Variable (based on temporal effects)
**Reads**: TemporalComponent, CausalityComponent, FunctionExecutorComponent
**Writes**: TemporalComponent.time_scale, CausalityComponent.causal_links
**Dependencies**: StabilitySystem

---

## Physics & Spatial Systems

### PhysicsSystem
**Responsibility**: Handles physics simulation and collision detection
**Update Frequency**: 60 FPS (fixed timestep)
**Reads**: PositionComponent, PhysicsComponent, BoundsComponent, TemporalComponent
**Writes**: PositionComponent, PhysicsComponent (velocity, acceleration)
**Dependencies**: TemporalSystem (for time-scaled physics)

### SpatialIndexSystem
**Responsibility**: Maintains spatial optimization structures (octrees, semantic graphs)
**Update Frequency**: Every 5 frames
**Reads**: PositionComponent, SemanticComponent, ProcessingLevelComponent
**Writes**: NetworkComponent (spatial connections), SemanticComponent (semantic_distance)
**Dependencies**: PhysicsSystem

### CollisionSystem  
**Responsibility**: Processes collision events and triggers
**Update Frequency**: Every frame
**Reads**: PositionComponent, BoundsComponent, PhysicsComponent
**Writes**: PhysicsComponent (collision response), HealthComponent (damage), EffectComponent (collision effects)
**Dependencies**: PhysicsSystem

---

## Matter & Voxel Systems

### VoxelSystem
**Responsibility**: Manages voxel world modification and rendering
**Update Frequency**: Every frame (high priority areas), Every 10 frames (low priority)
**Reads**: VoxelComponent, MatterComponent, FunctionExecutorComponent
**Writes**: VoxelComponent.voxel_data, MatterComponent (volume changes)
**Dependencies**: FunctionExecutionSystem

### MaterialInteractionSystem
**Responsibility**: Processes material reactions and transformations
**Update Frequency**: Every 3 frames
**Reads**: MatterComponent, MaterialInteractionComponent, TemporalComponent
**Writes**: MatterComponent (material changes), EffectComponent (reaction effects)
**Dependencies**: VoxelSystem

---

## Function & Reality Manipulation Systems

### FunctionExecutionSystem
**Responsibility**: Processes function execution and applies effects
**Update Frequency**: Every frame
**Reads**: FunctionCapabilityComponent, FunctionExecutorComponent, All relevant components
**Writes**: All components (based on function effects), EffectComponent
**Dependencies**: StabilitySystem, PhysicsSystem

### FunctionCompositionSystem
**Responsibility**: Handles function chaining, modifiers, and complex combinations
**Update Frequency**: Event-driven (on function activation)
**Reads**: FunctionExecutorComponent, FunctionCapabilityComponent
**Writes**: FunctionExecutorComponent (active functions), EffectComponent
**Dependencies**: FunctionExecutionSystem

### EffectManagementSystem
**Responsibility**: Manages temporary effects, duration tracking, and cleanup
**Update Frequency**: Every frame
**Reads**: EffectComponent, TemporalComponent
**Writes**: EffectComponent (duration updates), All affected components
**Dependencies**: TemporalSystem

---

## AI & Behavior Systems

### SensorySystem
**Responsibility**: Processes entity perception and awareness
**Update Frequency**: Every 3 frames (reduced for performance)
**Reads**: SensoryComponent, PositionComponent, All detectable components
**Writes**: AwarenessComponent, MemoryComponent (sensory memories)
**Dependencies**: SpatialIndexSystem

### AIBrainSystem
**Responsibility**: High-level AI decision making and behavior state management
**Update Frequency**: Every 5 frames per entity (staggered)
**Reads**: AIBrainComponent, AwarenessComponent, EmotionalComponent, HealthComponent
**Writes**: AIBrainComponent (behavior state), GOAPComponent (goal updates)
**Dependencies**: SensorySystem, EmotionalSystem

### GOAPSystem
**Responsibility**: Goal-Oriented Action Planning for intelligent entities
**Update Frequency**: Every 10 frames per entity (staggered)
**Reads**: GOAPComponent, All relevant world state components
**Writes**: GOAPComponent (current plan), AIBrainComponent (actions to execute)
**Dependencies**: AIBrainSystem

### FactionSystem
**Responsibility**: Manages faction relationships and group behaviors
**Update Frequency**: Every 30 frames
**Reads**: FactionComponent, MemoryComponent, AwarenessComponent
**Writes**: FactionComponent (reputation updates), AIBrainComponent (faction-influenced goals)
**Dependencies**: AIBrainSystem

---

## Memory & Consciousness Systems

### MemorySystem
**Responsibility**: Manages memory formation, decay, and retrieval
**Update Frequency**: Every 10 frames
**Reads**: MemoryComponent, EmotionalComponent, AwarenessComponent
**Writes**: MemoryComponent (memory updates), EmotionalComponent (memory-triggered emotions)
**Dependencies**: EmotionalSystem

### EmotionalSystem
**Responsibility**: Processes emotional states and their effects on behavior
**Update Frequency**: Every 5 frames
**Reads**: EmotionalComponent, HealthComponent, AwarenessComponent, MemoryComponent
**Writes**: EmotionalComponent, AIBrainComponent (emotion-influenced behavior)
**Dependencies**: SensorySystem

### ConsciousnessSystem
**Responsibility**: Manages awareness levels and conscious decision-making
**Update Frequency**: Every 10 frames
**Reads**: AwarenessComponent, MemoryComponent, EmotionalComponent, AIBrainComponent
**Writes**: AwarenessComponent (consciousness evolution), FunctionCapabilityComponent (awareness-gated abilities)
**Dependencies**: MemorySystem, EmotionalSystem

---

## Health & Combat Systems

### HealthSystem
**Responsibility**: Manages health, damage, and healing
**Update Frequency**: Every frame
**Reads**: HealthComponent, DamageComponent, EffectComponent
**Writes**: HealthComponent, Entity lifecycle (death/respawn)
**Dependencies**: CollisionSystem, EffectManagementSystem

### DamageSystem
**Responsibility**: Processes damage application and resistance calculations
**Update Frequency**: Event-driven (on damage events)
**Reads**: DamageComponent, HealthComponent, MatterComponent (armor)
**Writes**: HealthComponent, EffectComponent (damage effects)
**Dependencies**: HealthSystem

---

## World & Environment Systems

### WorldGenerationSystem  
**Responsibility**: Procedural generation of world chunks and zones
**Update Frequency**: Event-driven (on world loading)
**Reads**: SemanticComponent, ProcessingLevelComponent, WorldObjectComponent
**Writes**: All entity creation, VoxelComponent (terrain), MaterialInteractionComponent
**Dependencies**: SpatialIndexSystem

### WorldLoadingSystem
**Responsibility**: Dynamic loading/unloading of world areas based on semantic distance
**Update Frequency**: Every 30 frames
**Reads**: PositionComponent (player), SemanticComponent, ProcessingLevelComponent
**Writes**: ProcessingLevelComponent, Entity lifecycle (load/unload)
**Dependencies**: SpatialIndexSystem

### MemoryManifestationSystem
**Responsibility**: Spawns memory events and manifestations (director AI)
**Update Frequency**: Every 60 frames
**Reads**: MemoryComponent, EmotionalComponent, WorldState, ProcessingLevelComponent
**Writes**: Entity creation (memory manifestations), EffectComponent
**Dependencies**: MemorySystem, WorldGenerationSystem

---

## Interface & Interaction Systems

### InteractionSystem
**Responsibility**: Handles entity-to-entity interactions and player input
**Update Frequency**: Event-driven (on interaction attempts)
**Reads**: InteractableComponent, AwarenessComponent, FunctionCapabilityComponent
**Writes**: All components (based on interaction type)
**Dependencies**: SensorySystem, FunctionExecutionSystem

### ContainerSystem
**Responsibility**: Manages inventory and container contents
**Update Frequency**: Event-driven (on container access)
**Reads**: ContainerComponent, InteractableComponent
**Writes**: ContainerComponent, PositionComponent (item transfer)
**Dependencies**: InteractionSystem

---

## Performance & Optimization Systems

### ProcessingLevelSystem
**Responsibility**: Manages dynamic LOD and processing allocation
**Update Frequency**: Every 60 frames
**Reads**: ProcessingLevelComponent, PositionComponent, SemanticComponent, WorldState
**Writes**: ProcessingLevelComponent (load_factor updates)
**Dependencies**: SpatialIndexSystem, WorldStateSystem

### NetworkPropagationSystem
**Responsibility**: Handles effect and information propagation through semantic graphs
**Update Frequency**: Every frame (for active propagations)
**Reads**: NetworkComponent, EffectComponent, SemanticComponent
**Writes**: EffectComponent (propagated effects), NetworkComponent (connection strength)
**Dependencies**: SpatialIndexSystem, EffectManagementSystem

---

## System Execution Order

### Frame Update Sequence
```
1. WorldStateSystem (global state updates)
2. StabilitySystem (stability calculations)  
3. TemporalSystem (time effects)
4. PhysicsSystem (physics simulation)
5. CollisionSystem (collision processing)

// Parallel execution possible from here
6a. SensorySystem (perception)              | 6b. VoxelSystem (world modification)
7a. EmotionalSystem (emotional processing)  | 7b. MaterialInteractionSystem  
8a. MemorySystem (memory updates)           | 8c. FunctionExecutionSystem
9a. AIBrainSystem (AI decisions)            | 9d. EffectManagementSystem

// Sequential execution resumes
10. HealthSystem (health/damage processing)
11. InteractionSystem (player/entity interactions)
12. SpatialIndexSystem (spatial optimization)
13. ProcessingLevelSystem (LOD updates)
```

### Periodic Updates (Lower Frequency)
```
Every 5 frames: GOAPSystem (AI planning)
Every 10 frames: ConsciousnessSystem, WorldGenerationSystem
Every 30 frames: FactionSystem, WorldLoadingSystem  
Every 60 frames: MemoryManifestationSystem
```

### Event-Driven Systems
- FunctionCompositionSystem (on function activation)
- DamageSystem (on damage events)
- InteractionSystem (on interaction attempts)
- ContainerSystem (on container access)

## Performance Characteristics

### High-Frequency Systems (Every Frame)
- WorldStateSystem, StabilitySystem, PhysicsSystem, CollisionSystem
- FunctionExecutionSystem, EffectManagementSystem, HealthSystem
- **Total**: ~8 systems per frame

### Medium-Frequency Systems (Every 3-10 Frames)  
- SensorySystem, EmotionalSystem, MemorySystem, AIBrainSystem
- VoxelSystem, MaterialInteractionSystem
- **Load Balancing**: Staggered updates across entities

### Low-Frequency Systems (Every 30+ Frames)
- FactionSystem, WorldLoadingSystem, ProcessingLevelSystem
- MemoryManifestationSystem
- **Purpose**: Strategic/environmental management

### Parallelization Opportunities
Systems with non-overlapping write components can run in parallel:
- **Group A**: Sensory, Emotional, Memory, AI (entity state)
- **Group B**: Voxel, Material, Function, Effect (world state)
- **Synchronization Point**: Before HealthSystem and InteractionSystem

This system architecture ensures predictable data flow while enabling complex emergent behaviors through system interactions.
