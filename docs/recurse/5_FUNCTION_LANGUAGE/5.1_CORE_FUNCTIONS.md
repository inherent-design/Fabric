# RECURSE Core Functions
*The Essential 12 Reality Manipulation Primitives*

Core Functions are the fundamental "verbs" of reality manipulation. This document defines the minimized set of 12 essential functions that provide the foundation for all emergent gameplay in RECURSE.

---

## Design Philosophy

**Minimalism with Emergence**: Each function is a simple primitive that gains complexity through combination with modifiers and other functions. No function is redundant - each fills a unique role in the reality manipulation language.

**Cost-Benefit Balance**: Every function has clear energy and stability costs that create meaningful resource management decisions.

**Semantic Clarity**: Function names and behaviors are intuitive, allowing players to predict outcomes and plan combinations.

---

## Matter Manipulation (3 Functions)

### 1. MATTER_SOURCE
**Purpose**: Create matter from energy
```gdscript
Parameters:
    material_type: MaterialType
    amount: float           # Volume in cubic meters
    target_position: Vector3
    
Base Costs:
    energy_cost: 8.0 * amount * material_density_modifier
    stability_cost: 0.04 * amount
    
Material Density Modifiers:
    ROCK: 1.0, WATER: 0.8, METAL: 1.5, AIR: 0.1, PLASMA: 2.0
```

### 2. MATTER_TRANSFORM  
**Purpose**: Change material properties of existing matter
```gdscript
Parameters:
    target: EntityID or Area
    source_material: MaterialType
    target_material: MaterialType
    transformation_rate: float    # Cubic meters per second
    
Base Costs:
    energy_cost: 6.0 * volume * transformation_difficulty
    stability_cost: 0.06 * volume
    
Transformation Difficulty:
    Same category (solid->solid): 1.0
    Phase change (solid->liquid): 1.5  
    Exotic change (matter->plasma): 3.0
```

### 3. MATTER_DISSOLVE
**Purpose**: Deconstruct matter back into energy
```gdscript
Parameters:
    target: EntityID or Area
    dissolution_rate: float       # Cubic meters per second
    
Base Costs:
    energy_cost: 4.0 * volume    # Returns 2.0 energy per volume
    stability_cost: 0.08 * volume
    
Net Energy: -2.0 * volume (positive energy gain)
```

## Energy & Force (3 Functions)

### 4. KINETIC_FORCE
**Purpose**: Apply directional force to objects
```gdscript
Parameters:
    target: EntityID or Position
    force_vector: Vector3
    force_magnitude: float        # Newtons
    
Base Costs:
    energy_cost: 1.5 + (force_magnitude / 100.0)
    stability_cost: 0.01 + (force_magnitude / 1000.0)
```

### 5. ENERGY_BARRIER
**Purpose**: Create protective energy fields
```gdscript
Parameters:
    position: Vector3
    size: Vector3                 # Dimensions
    absorption_capacity: float    # Total energy it can absorb
    duration: float               # Seconds, 0 = permanent until destroyed
    
Base Costs:
    energy_cost: 12.0 + (absorption_capacity / 10.0)
    stability_cost: 0.02 * duration (capped at 2.0 for permanent)
```

### 6. GRAVITY_WELL
**Purpose**: Create localized gravitational anomalies  
```gdscript
Parameters:
    center_position: Vector3
    radius: float
    gravity_multiplier: float     # 1.0 = normal, 0.0 = zero-g, -1.0 = repulsion
    duration: float
    
Base Costs:
    energy_cost: 20.0 + (abs(gravity_multiplier - 1.0) * 8.0)
    stability_cost: 0.12 + (abs(gravity_multiplier - 1.0) * 0.05)
```

## Spacetime (3 Functions)

### 7. TIME_DILATION
**Purpose**: Alter local time flow
```gdscript
Parameters:
    target_area: Sphere or Box
    time_factor: float            # 0.5 = half speed, 2.0 = double speed
    duration: float               # Real-time seconds
    
Base Costs:
    energy_cost: 25.0 + (abs(time_factor - 1.0) * 15.0)
    stability_cost: 0.15 + (abs(time_factor - 1.0) * 0.1)
```

### 8. SPATIAL_PORTAL
**Purpose**: Create connection between two points in space
```gdscript
Parameters:
    entry_point: Vector3
    exit_point: Vector3
    radius: float                 # Portal size
    duration: float
    
Base Costs:
    energy_cost: 35.0 + (distance / 10.0) + (radius * 5.0)
    stability_cost: 0.2 + (distance / 100.0)
```

### 9. SPACE_DISTORTION
**Purpose**: Warp spatial geometry in localized area
```gdscript
Parameters:
    center: Vector3
    radius: float
    distortion_type: String       # "EXPAND", "COMPRESS", "TWIST", "LENS"
    intensity: float              # How much to distort (0.0-2.0)
    duration: float
    
Base Costs:
    energy_cost: 18.0 + (intensity * 12.0)
    stability_cost: 0.14 + (intensity * 0.08)
```

## Information (3 Functions)

### 10. DATA_SENSE
**Purpose**: Gather information about entities and environment
```gdscript
Parameters:
    target: EntityID, Area, or "ALL_NEARBY"
    data_type: String             # "MATERIAL", "HEALTH", "FUNCTION", "MEMORY", "WEAKNESS"
    scan_depth: float             # How detailed the scan is
    
Base Costs:
    energy_cost: 0.5 + (scan_depth * 2.0)
    stability_cost: 0.001         # Minimal stability impact
```

### 11. MEMORY_PROBE  
**Purpose**: Access historical/memory data from objects and locations
```gdscript
Parameters:
    target: EntityID or Position
    time_depth: float             # How far back to look (seconds)
    memory_type: MemoryType       # Filter for specific memory types
    
Base Costs:
    energy_cost: 8.0 + (time_depth / 60.0)  # More expensive for older memories
    stability_cost: 0.03 + (time_depth / 1200.0)
```

### 12. SYSTEM_OVERRIDE
**Purpose**: Take temporary control of non-sentient systems
```gdscript
Parameters:
    target_system: EntityID
    override_command: String      # "ACTIVATE", "DEACTIVATE", "REVERSE", "CUSTOM"
    duration: float
    
Base Costs:
    energy_cost: 10.0 + (system_complexity * 3.0)
    stability_cost: 0.025 * duration
    
System Complexity:
    Simple (doors, lights): 1.0
    Moderate (machinery): 2.0  
    Complex (security systems): 4.0
```

## Function Synergies & Combinations

### Common Two-Function Combinations
- **MATTER_SOURCE + KINETIC_FORCE**: Basic projectile launcher
- **ENERGY_BARRIER + TIME_DILATION**: Extended-duration shields  
- **SPATIAL_PORTAL + SYSTEM_OVERRIDE**: Remote system control
- **MATTER_DISSOLVE + ENERGY_BARRIER**: Self-powered barriers
- **MEMORY_PROBE + DATA_SENSE**: Complete environmental analysis
- **GRAVITY_WELL + SPACE_DISTORTION**: Reality warping fields

### Three-Function Combinations
- **MATTER_SOURCE + MATTER_TRANSFORM + KINETIC_FORCE**: Custom ammunition system
- **TIME_DILATION + ENERGY_BARRIER + GRAVITY_WELL**: Temporal defense bubble
- **DATA_SENSE + MEMORY_PROBE + SYSTEM_OVERRIDE**: Perfect infiltration tool

### Advanced Combinations (4+ Functions)
- **Complete Matter Recycling**: MATTER_DISSOLVE → MATTER_SOURCE → MATTER_TRANSFORM → KINETIC_FORCE
- **Temporal Construction**: TIME_DILATION + MATTER_SOURCE + SPATIAL_PORTAL + ENERGY_BARRIER

## Mastery & Progression System

### Function Mastery Levels
```gdscript
Novice (0.0-0.25):      Base costs, standard effects
Apprentice (0.25-0.50): 10% cost reduction, minor effect improvements  
Adept (0.50-0.75):      20% cost reduction, moderate effect improvements
Expert (0.75-0.90):     30% cost reduction, major effect improvements
Master (0.90-1.0):      40% cost reduction, unique mastery effects
```

### Mastery Benefits Examples
- **MATTER_SOURCE Mastery**: Unlock exotic materials, create complex shapes
- **TIME_DILATION Mastery**: Affect larger areas, create time loops
- **MEMORY_PROBE Mastery**: Access suppressed memories, implant false memories
- **SYSTEM_OVERRIDE Mastery**: Control sentient entities temporarily

### Unlocking New Functions
Functions are discovered through:
1. **Environmental Scanning**: Using DATA_SENSE on anomalous phenomena
2. **Memory Archaeology**: MEMORY_PROBE on significant locations  
3. **Experimental Combination**: Successfully combining functions in novel ways
4. **NPC Interaction**: Learning from other reality manipulators

## Resource Management

### Energy System
- **Base Pool**: 100 energy points
- **Regeneration**: 2.0 energy per second (base rate)
- **Overcharge Penalty**: Using functions above energy capacity causes health damage
- **Efficiency Scaling**: Higher mastery = lower energy costs

### Stability Management  
- **Local Stability**: Each area has independent stability (0.0-1.0)
- **Stability Regeneration**: 0.02 per second when no functions are active
- **Cascade Failure**: Low stability areas can trigger chain reactions
- **Anchor Points**: Some locations provide stability bonuses

### Strategic Considerations
- **Function Ordering**: Some combinations are more efficient in specific orders
- **Cooldown Management**: Functions have brief cooldowns to prevent spam
- **Risk/Reward**: More powerful effects require higher stability costs
- **Environmental Factors**: Some materials/areas modify function effectiveness
