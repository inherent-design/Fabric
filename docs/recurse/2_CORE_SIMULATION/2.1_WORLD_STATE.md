# RECURSE World State Enums & State Machines
*The Fundamental Language of Reality Simulation*

This document defines the core enums, state machines, and data types that form the foundation of RECURSE's reality simulation. Every system in the game operates using these shared primitives.

## Reality Stability System

```gdscript
enum StabilityLevel {
    STABLE,      # 1.0 - 0.8: Functions work as expected
    WAVERING,    # 0.7 - 0.5: Minor side effects possible
    UNSTABLE,    # 0.4 - 0.2: Major side effects, function failures
    FRACTURED    # 0.1 - 0.0: Reality breakdown, catastrophic events
}

enum StabilityEvent {
    FUNCTION_USAGE,        # Using reality manipulation functions
    MATTER_CREATION,       # Creating matter from energy
    TEMPORAL_MANIPULATION, # Time dilation effects
    SPATIAL_DISTORTION,    # Warping space
    MEMORY_MANIFESTATION,  # Memories becoming real
    GLITCH_INFECTION,      # External corruption spreading
    ANCHOR_PLACEMENT,      # Stabilizing interventions
    NATURAL_REGENERATION   # Passive stability recovery
}

# State Machine: Stability Transitions
# STABLE -> WAVERING (0.1 stability loss threshold)
# WAVERING -> UNSTABLE (0.3 additional loss)
# UNSTABLE -> FRACTURED (0.2 additional loss)
# Recovery is slower: 0.02/second natural regeneration
```

## State Machines

### Reality Stability State Machine
```
STABLE ──(function_usage/0.1)──> WAVERING
   ↑                                │
   │                                │
(regeneration/0.02s)              │
   │                                ↓
   └── FRACTURED <──(0.2)─── UNSTABLE
        │    ↑           ↑
        │    └───(0.5)───┘
        │
        └──(cascade_failure)──> VOID
```

### NPC Awareness State Machine
```
UNAWARE ──(sense_stimulus)──> ALERT
   ↑                            │
   │                            │
(timeout/10s)                  │
   │                            ↓
   └─── SEARCHING <──(lost_target)── PURSUING
             │                        ↑
             └──(found_target)────────┘
```

### Function Execution State Machine
```
READY ──(execute)──> PROCESSING ──(complete)──> COOLDOWN
  ↑                      │                        │
  │                      │                        │
  │                      ↓                        │
  │                   FAILED                      │
  │                      │                        │
  └──(timeout)───────────┴────(timeout)──────────┘
```

## Core Data Structures

```gdscript
# World State Component
class WorldState:
    var stability: float = 1.0
    var stability_level: StabilityLevel = StabilityLevel.STABLE
    var time_scale: float = 1.0
    var active_effects: Array[GameEffect] = []

# Memory Fragment
class MemoryFragment:
    var memory_type: MemoryType
    var emotional_impact: float
    var clarity: float          # How clear/accessible the memory is
    var timestamp: float
    var associated_location: Vector3
    var trigger_conditions: Dictionary

# Causal Link
class CausalLink:
    var cause_entity: EntityID
    var effect_entity: EntityID
    var strength: CausalityStrength
    var delay: float
    var conditions: Array[String]

# Function Blueprint
class FunctionBlueprint:
    var function_type: FunctionType
    var modifiers: Array[FunctionModifier]
    var parameters: Dictionary
    var execution_type: ExecutionType
    var energy_cost: float
    var stability_cost: float
```

---

This enumeration system provides the shared vocabulary for all RECURSE systems. Every component, system, and gameplay mechanic operates using these defined types, ensuring consistency and enabling complex emergent interactions between systems.

The state machines define the rules for how these enum values transition, creating predictable yet dynamic behavior throughout the game world.
